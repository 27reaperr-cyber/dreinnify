# 🎵 Dreinnify — Telegram-бот + Mini-App (Node.js, один файл)

Полностью переписанный проект на **Node.js**. Весь код — в одном файле `index.js`,
который одновременно поднимает **Telegram-бота** и **Mini-App** (веб-приложение).
Подключение Mini-App к домену настраивается через `.env`.

Источник музыки — **YouTube Music** (поиск + полноценные аудио-потоки, без 30-секундных превью).

---

## ⚙️ Возможности

**Дизайн Mini-App**
- Современный тёмный интерфейс в стиле Spotify, шрифт **Manrope** везде.
- Полная адаптивность: на ПК — боковое меню, на мобильных — меню снизу.
- Лёгкая вибрация (**haptic feedback**) с возможностью отключения в настройках.

**Плеер и контент**
- Треки, альбомы, обложки и авторы, включая **фиты** (приглашённых артистов).
- **Синхронизированный текст** (караоке): строки двигаются и подсвечиваются в такт музыке (через lrclib.net).
- На главной — промо-блок **«Выбрано редакцией»** и список до 10 треков.
- Функция **«Моя волна» полностью удалена** (как и требовалось).

**Социальные функции**
- Плейлисты (создание, добавление/удаление треков, удаление плейлиста).
- Лайки треков и альбомов — сохраняются в профиле.
- Кастомные профили: имя, описание (bio), аватар.
- Система **рецензий** до **90 баллов** = сумма 6 критериев (каждый 1–5) × 3:
  Редкость · Целостность · Глубина · Реализация · Харизма/Атмосфера · Актуальность.

**Админ-панель** (только для `ADMIN_ID`)
- Бан / разбан пользователей.
- Статистика бота.
- Верификация (синяя галочка в профиле).
- Статус автора (иконка ноты на фиолетовом фоне).
- Управление блоком «Выбрано редакцией»: добавление/удаление треков и вкл/выкл блока.

## 🚀 Установка и запуск

```bash
# 1. Установить зависимости
npm install

# 2. Создать .env на основе примера
cp .env.example .env
#    и заполнить BOT_TOKEN, ADMIN_ID, MINIAPP_DOMAIN

# 3. Запустить
npm start
```

Требуется **Node.js 18+** (используется встроенный `fetch`).

## 🔧 Переменные окружения (.env)

| Переменная | Описание |
|---|---|
| `BOT_TOKEN` | Токен Telegram-бота от @BotFather |
| `ADMIN_ID` | Telegram ID администратора (числовой) |
| `MINIAPP_DOMAIN` | Публичный **HTTPS**-адрес, где доступно приложение (например `https://music.example.com`) |
| `PORT` | Порт веб-сервера (по умолчанию `3000`) |
| `DB_PATH` | Путь к файлу базы SQLite (по умолчанию `./music.db`) |
| `DEV_MODE` | `1` — тестировать Mini-App в браузере без Telegram (фиктивный пользователь) |

> **Важно:** Telegram требует, чтобы Mini-App открывался по **HTTPS**.
> Для продакшена поставьте reverse-proxy (nginx / Caddy) перед приложением, либо
> используйте туннель (cloudflared / ngrok) и укажите его адрес в `MINIAPP_DOMAIN`.
> После запуска пропишите этот же адрес боту в @BotFather → *Bot Settings → Menu Button* (необязательно — бот выставит кнопку сам).

## 🏗️ Как это работает

- **Express** отдаёт страницу Mini-App на `/` и REST API на `/api/*`.
- Авторизация Mini-App — по подписи Telegram `initData` (HMAC-SHA256), проверяется на сервере.
- **YouTube Music** (`ytmusic-api`) — поиск треков/альбомов и метаданные.
- **@distube/ytdl-core** — потоковая отдача аудио целиком на `/api/stream/:id`.
- **lrclib.net** — синхронизированный текст по тайм-кодам.
- **better-sqlite3** — хранение пользователей, треков, плейлистов, лайков, рецензий и настроек.
- **Telegraf** — Telegram-бот (`/start` с кнопкой запуска Mini-App, `/app`, `/stats`).
