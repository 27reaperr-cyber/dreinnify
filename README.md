# 🎧 Musicbott v2.0

**Один файл `bot.js`** = Telegram-бот **+** Mini-App (Spotify-style, тёмная тема, Manrope).
Использует **YouTube Music** через `yt-dlp` (без 30-секундных ограничений Deezer).

---

## 📦 Что внутри

| Компонент      | Описание                                                                 |
|----------------|--------------------------------------------------------------------------|
| **bot.js**     | Единственный исходник. Запускает бота (`node-telegram-bot-api`) + HTTP-сервер Mini-App (`express`). |
| **package.json** | Зависимости.                                                          |
| **.env.example** | Шаблон конфигурации.                                                  |

Mini-App-фронтенд (HTML/CSS/JS, ~75 КБ) встроен в `bot.js` через base64 — никаких отдельных static-файлов не требуется.

---

## 🛠 Требования

| Пакет        | Зачем                                  | Установка                        |
|--------------|----------------------------------------|----------------------------------|
| **Node.js ≥ 18** | runtime                            | https://nodejs.org               |
| **yt-dlp**   | поиск и стриминг YouTube Music         | `pip install -U yt-dlp` или [бинарник](https://github.com/yt-dlp/yt-dlp/releases) |
| **ffmpeg**   | (опционально) для будущих эффектов     | `apt install ffmpeg` / `brew install ffmpeg` |

Бот сам найдёт `yt-dlp` и `ffmpeg` в PATH.

## ⚡ Быстрый старт

```bash
# 1) Установка зависимостей
npm install

# 2) Настройка
cp .env.example .env
# → отредактировать BOT_TOKEN, MINI_APP_URL, ADMIN_ID

# 3) Запуск
node bot.js
```

## 🌐 Подключение Mini-App к домену

1. У вас должен быть **публичный HTTPS-домен** с валидным сертификатом, указывающий на ваш сервер (порт из `.env`).
   Самый простой вариант — поставить **nginx** или **Caddy** в режиме reverse-proxy перед Node.js.
2. В `.env` пропишите:
   ```
   MINI_APP_URL=https://music.example.com
   ```
3. Откройте **@BotFather** → ваш бот → **Bot Settings** → **Menu Button** → **Configure Menu Button** → введите тот же URL.
   (Дополнительно можно через `/newapp` зарегистрировать его как Mini-App.)
4. Готово — кнопка `🎧 Открыть плеер` в `/start` откроет ваше приложение.

> ⚠️ Telegram требует **HTTPS**. Бот при старте откажется работать с HTTP-URL.

### Пример минимального Caddy-конфига:
```
music.example.com {
    reverse_proxy localhost:3000
}
```

## ✨ Возможности

### Mini-App
- **Spotify-style** тёмная тема, шрифт **Manrope**.
- Адаптив: на ПК — sidebar, на мобиле — нижний нав-бар (как в Spotify).
- **Haptic feedback** (вибрация) — настраивается в Настройках.
- Главная: **«Выбрано редакцией»** + список топ-треков (до 10).
- **Поиск** YouTube Music с автодополнением.
- **Плеер** с очередью, shuffle, repeat (off/all/one), регулировкой громкости.
- **Synchronized Lyrics** — режим караоке через [lrclib.net](https://lrclib.net) (бесплатно, без ключа).
- **Лайки** треков, сохраняются в раздел «Любимое».
- **Плейлисты** — создание, переименование, удаление, добавление треков.
- **Кастомные профили** — био, цвет акцента.
- **Рецензии** до 90 баллов (6 критериев × 5 баллов × 3):
  Редкость · Целостность · Глубина · Реализация · Харизма · Актуальность.
- **Featured-артисты** определяются из заголовка трека.

### Бот
- `/start` — приветствие + кнопка `WebApp`.
- `/menu` — повторно показать кнопку.
- `/admin` — краткая статистика (только для ADMIN_ID).

### Админ-панель (внутри Mini-App, доступна только ADMIN_ID)
- 📊 Мониторинг статистики (пользователи, треки, прослушивания, рецензии).
- 👥 Список пользователей: бан / разбан / верификация (синяя ✓) / статус автора (фиолетовая нота).
- ⭐ Управление блоком **«Выбрано редакцией»**: добавить трек (через поиск), удалить, **включить/выключить** весь блок.

## 🗄 База данных

Файл `bot.db` (SQLite, WAL-режим) создаётся автоматически рядом с `bot.js`. Кеш аудио — в `./cache/`.

## 📁 Структура файлов (после запуска)

```
.
├── bot.js              ← основной исходник (всё внутри)
├── package.json
├── .env                ← ваш конфиг
├── bot.db              ← SQLite
├── cache/              ← кеш аудио (опционально)
└── node_modules/
```

## 🐛 Известные нюансы

- Прямые аудио-URL от YouTube Music живут ~6 часов; бот кеширует их на 25 минут и автоматически обновляет.
- Если стрим внезапно перестал работать — `yt-dlp` устарел: обновите `pip install -U yt-dlp`.
- LRCLib не покрывает все песни — если синхротекст не нашёлся, в плеере покажется заглушка или обычный текст.
